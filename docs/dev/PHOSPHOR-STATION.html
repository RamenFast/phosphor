📺 Phosphor

Rust-native oscilloscope / beam visualizer — winit + wgpu + egui, pipewire capture, six themes, blossom default. Waves 1–3.2 shipped and daily-drivable. The agent CLI is live: phosphor speaks the station convention natively — probe, ctl, tap, feed, kit, schema.

What runs today

phosphor                    # the instrument (GUI); --mini, --screensaver, --visitor
phosphor --background       # same instrument on a private Xvfb display: renders, serves
                            # the socket/tap/feed, never maps a window or steals focus
phosphor render in out.mp4  # headless track render
phosphor bench              # perf receipts
phosphor --version

phosphor probe --json       # live status one-shot (running:false when no GUI)
phosphor ctl mode ring      # drive the running instrument over the control socket
phosphor tap | jq .         # NDJSON stream of beam frames (hello, frame, tick)
phosphor kit validate k.phoskit   # validate/inspect a kit without launching the GUI
phosphor schema             # strict JSON schemas + enums, self-description

The agent contract — shipped

The subcommand surface was spec'd agent-grade in crates/phosphor-app/src/main.rs: --json, exit codes 0/2/3/4 — the same scheme the station convention adopted. Wave 3.2 built it out. What the verbs do:

verbdoesstatus
probelive status one-shot (--json); running:false when no GUIshipped
ctlcontrol-socket verbs: play/pause/mode/theme/capture/snapshot/clip/quit…shipped
tapNDJSON beam stream — hello, frame, tick heartbeatshipped
feedthe locked v3-verbatim applet protocol (engine-free applet draws from it)shipped
kitvalidate/inspect visual kits + shipped JSON schemasshipped
studiothe composition surfacepending — wave 4

Bedtime law on record, now kept: a 7B model repairs its kit in one round-trip. Phosphor is the station's visual-instrument node — probe --at a past timestamp is the one stub that waits for the studio wave.

The seat is taken

The contract shipped exactly as designed, with zero convention retrofit — the envelope {status,tool,version,ts}, fix-bearing errors (every error carries the fix), exit codes 0/2/3/4, --json forcing (isatty auto-switches otherwise), NDJSON with a hello greeting and tick heartbeat, and phosphor schema self-description (strict schemas + enums, mirrored to docs/phoskit.schema.json). The one documented exception: feed keeps its locked v3-verbatim applet protocol — no event field, on purpose, so the engine-free applet stays byte-compatible.
